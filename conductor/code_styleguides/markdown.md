# Markdown Style Guide
- **Artifact-First:** Markdown files are functional artifacts for agents. Use strict formatting.
- **Headers:** Use ATX-style headers (#, ##). Maintain a clear hierarchy.
- **Lists:** Use - for unordered lists and 1. for ordered lists. Use [ ] for task tracking.
- **Code Blocks:** Always specify the language for syntax highlighting and parsing.
- **Parsability:** Avoid nested complex structures that break regex-based parsing.
