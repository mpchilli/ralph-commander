# Git Commit Style Guide
- **Snapshot Protocol:** Use CAPTAIN_SNAPSHOT: [task_id] for automated atomic snapshots.
- **Semantic Commit Messages:** Use conventional commits (eat:, ix:, docs:) for human-readable changes.
- **Auditability:** Include the 'Mission Control' task ID in the footer of implementation commits.
- **Sovereignty:** Semantic version tags must be triggered via the Release Gate protocol.
